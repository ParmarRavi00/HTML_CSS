2). What are tags and attributes in HTML?
Ans. Tags are used to define different elements in an HTML document, such as headings, paragraphs, links, images, and more. Each HTML tag is enclosed within angle brackets and typically consists of a starting tag and an ending tag. For example, the <h1> tag is used to define a top-level heading, and it's enclosed within <h1> and </h1> tags.

Attributes, on the other hand, provide additional information about an HTML element. They are specified within the starting tag of an element and are used to modify the behavior or appearance of an element. For example, the <a> tag is used to define a hyperlink, and it has an href attribute that specifies the URL that the link points to. Another example is the <img> tag used to display images, it has attributes such as src to specify the image file URL, alt to provide a text description of the image, width and height to control the image dimensions, among others.

In summary, HTML tags define the structure of an HTML document, and attributes provide additional information to the tags to help define the behavior and appearance of elements.

3). What are void elements in HTML?
Ans. Void elements in HTML are elements that do not have a closing tag and are self-closing. They are also known as empty elements. Void elements are used to insert an element into an HTML page without any content or child elements.

Some examples of void elements in HTML include:

<img> tag for images
<br> tag for line breaks
<input> tag for form inputs
<meta> tag for metadata
<link> tag for linking external resources
<hr> tag for horizontal rule 
Void elements are written using a special syntax with a forward slash at the end of the tag, like <img src="image.png" alt="An image"/>. It's worth noting that while some HTML elements are always void elements (such as <br>), other elements can be void in certain contexts, depending on the HTML version being used.

Void elements do not have any content or child elements, so their purpose is to provide additional information or functionality to the HTML document. They are important for web developers because they allow the creation of well-formed and valid HTML code, and are supported by all modern web browsers.

4). What are HTML Entities? 
Ans. HTML entities are special characters that are used in HTML to display text or symbols that would otherwise not be displayed by the browser. This is because some characters, such as angle brackets, ampersands, and other special characters, have special meanings in HTML and are used to define the structure of an HTML document.

HTML entities are specified using a specific syntax that begins with an ampersand (&) and ends with a semicolon (;). For example, the HTML entity for the less than symbol (<) is "<", and the HTML entity for the greater than symbol (>) is ">". Other common HTML entities include:

& for the ampersand symbol (&)
" for the double quote symbol (")
' for the single quote symbol (')
  for a non-breaking space
Using HTML entities is important because it allows developers to display special characters or symbols without confusing the browser or causing errors in the HTML document. It is also necessary when displaying non-ASCII characters, such as those used in foreign languages or mathematical symbols, as they may not be recognized by all browsers.

5). What are different types of lists in HTML?
Ans. In HTML, there are three main types of lists: ordered lists, unordered lists, and definition lists.

--> Ordered Lists:
An ordered list is used to create a numbered list of items, which are typically displayed in a sequence. To create an ordered list, the <ol> tag is used, and each item in the list is defined using the <li> tag.

--> Unordered Lists:
An unordered list is used to create a bulleted list of items, where each item is displayed with a bullet point. To create an unordered list, the <ul> tag is used, and each item in the list is defined using the <li> tag.

--> Definition Lists:
A definition list is used to create a list of terms and their corresponding definitions. To create a definition list, the <dl> tag is used, and each term in the list is defined using the <dt> tag, while the corresponding definition is defined using the <dd> tag. 

Using these types of lists in HTML allows web developers to organize content in a clear and meaningful way, making it easier for users to read and understand.
